# Checklist de Validação UX — WhatFlow

Use este checklist ao revisar os wireframes antes de começar o Sprint 1.

## Dashboard

- [ ] KPIs fazem sentido para o negócio?
- [ ] Gráfico de volume de mensagens está claro?
- [ ] Status das instâncias é visível de imediato?
- [ ] Seletor de período é acessível?
- [ ] Seletor de instância permite filtrar por conexão?

## Inbox (Caixa de Entrada)

- [ ] Layout 3 colunas funciona na tela disponível?
- [ ] Toggle Bot ↔ Humano está em lugar óbvio?
- [ ] Filtros de status (Abertas, Pendentes, Minhas) são úteis?
- [ ] Badge de não lidos é visível?
- [ ] Painel do contato (coluna 3) tem informações suficientes?
- [ ] Ações rápidas (Resolver, Atribuir, Bloquear) estão acessíveis?
- [ ] Notas internas são claramente diferentes de mensagens?

## Flow Builder

- [ ] Paleta de nós está acessível e organizada por categoria?
- [ ] Canvas com exemplo é intuitivo para entender o fluxo?
- [ ] Painel de propriedades tem todos os campos necessários?
- [ ] Botão "Publicar" está em destaque?
- [ ] Badge de status (Rascunho / Publicado) é visível?
- [ ] Insertor de variáveis ({{nome}}, etc.) está claro?

## Geral

- [ ] Navegação lateral (sidebar) faz sentido?
- [ ] Identidade visual (cores, tipografia) está adequada?
- [ ] Layout responsivo seria um requisito para este MVP?

## Resultado

Se < 80% dos itens aprovados: revisar wireframes antes de codar.
Se ≥ 80%: avançar para o Passo 2 (Supabase).
